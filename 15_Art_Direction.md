# 15 — Art Direction

## Purpose

This document defines the visual identity of the portfolio.

It is not a branding guide.

It is an interaction guide.

Every visual decision should reinforce Alejandro's way of thinking:

Calm.

Structured.

Curious.

Intentional.

Timeless.

---

# Visual Statement

The interface should feel like entering a carefully organized creative workspace.

Not an advertisement.

Not a landing page.

Not a personal résumé.

Not an art gallery.

It should feel like a place where ideas are explored, connected and refined.

---

# Emotional Palette

The visual language should communicate:

Curiosity.

Trust.

Precision.

Depth.

Craftsmanship.

Continuous improvement.

Never:

Urgency.

Aggression.

Luxury.

Entertainment.

Excess.

---

# Color Philosophy

Color communicates hierarchy before decoration.

The portfolio should never depend on color to attract attention.

Instead, color should quietly support navigation and understanding.

The interface must remain elegant even when viewed in grayscale.

If it only works because of color, it is poorly designed.

---

# Primary Identity

# Contrast

Accessibility is mandatory.

Every text combination must satisfy WCAG AA.

Whenever possible:

WCAG AAA.

Beauty never justifies poor readability.

---

# Materials

Avoid glassmorphism.

Avoid heavy gradients.

Avoid noisy textures.

Instead use:

Soft surfaces.

Subtle depth.

Thin shadows.

Light transitions.

Small reflections.

The interface should feel tangible.

Not glossy.

---

# Light

The interface should appear naturally illuminated.

Not artificially lit.

Highlights should suggest ambient light.

Not stage lighting.

---

# Shadows

Shadows communicate elevation.

Not decoration.

Small shadows.

Soft edges.

Large blur radius.

Low opacity.

Never dramatic.

---

# Corners

Rounded.

Never exaggerated.

Consistent radius throughout the interface.

The interface should feel engineered.

Not playful.

---

# Typography Color

Titles

Highest contrast.

Body

Slightly softer.

Captions

Muted.

Metadata

Even lighter.

The visual hierarchy should emerge naturally.

Not through large size differences.

---

# Photography

Photography should feel authentic.

Natural light.

Minimal editing.

No corporate poses.

No fake smiles.

No stock aesthetics.

Portraits should integrate into the interface rather than appearing as isolated rectangles.

Whenever possible, backgrounds should visually merge with the page.

The portrait should feel like another layer of the composition.

---

# Project Images

Every project cover should follow the same visual language.

Large margins.

Consistent perspective.

Consistent lighting.

Minimal text.

No marketing banners.

No decorative mockups.

Projects should look documented.

Not advertised.

---

# Icons

Thin strokes.

Rounded corners.

Minimal detail.

Consistent optical weight.

Icons support reading.

They never replace it.

---

# Motion Relationship

Color and motion should reinforce each other.

Hover states should subtly increase saturation.

Active states should slightly deepen the primary color.

Transitions should never flash.

---

# Visual Density

The interface should breathe.

Negative space is intentional.

Objects should never compete.

Every section should have a clear focal point.

---

# What We Never Do

No neon colors.

No rainbow gradients.

No glowing buttons.

No glass everywhere.

No RGB effects.

No cyberpunk aesthetics.

No fake 3D.

No unnecessary visual noise.

No decorative backgrounds.

No trendy effects that age quickly.

---

# Final Principle

If the interface still feels modern five years from now, the design has succeeded.

If visitors remember clarity before colors, the visual language is working.
