# Screen 20 — Home

## Purpose

The Home screen is not an introduction.

It is an invitation.

Visitors should immediately understand that this portfolio is different without needing to read large amounts of text.

Curiosity must appear before information.

---

# Emotional Goal

The visitor should think:

"What is this?"

Five seconds later:

"I want to keep exploring."

---

# Visual Identity

The Home screen should feel spacious.

Minimal.

Calm.

Confident.

Nothing fights for attention.

The page breathes.

Large margins.

Large empty areas.

The interface should communicate confidence through restraint.

---

# Layout

Viewport height

100vh

No scrolling required to understand the first screen.

Everything important should exist above the fold.

---

# Composition

Navigation

Top.

Minimal.

Transparent.

Logo

None.

The portfolio itself is the identity.

Language Switch

Top right.

Theme Switch

Beside language.

Subtle.

Never dominant.

---

# Hero

Portrait

Left side.

Integrated into the background.

Never inside a rectangle.

The portrait should visually merge with the interface.

Soft lighting.

Neutral expression.

Looking slightly toward the center of the page.

---

# Portrait Placeholder

Expected asset:

Transparent PNG or WebP

Approx. 1800 px height

Soft edge masking

Neutral clothing

Clean background

Natural side lighting

No artificial poses

Expression:

Curious.

Calm.

Approachable.

---

# Headline

Largest element on screen.

Maximum two lines.

Visitors should read it in under three seconds.

---

# Subheadline

Three to four lines.

Readable.

Never marketing language.

---

# CTA

Primary

Explore

Secondary

GitHub

Resume

LinkedIn

---

# Three.js Scene

The background contains an abstract network.

Not particles.

Not stars.

Nodes.

Connections.

Relationships.

Nothing random.

The scene reacts slightly to mouse movement.

Movement remains subtle.

Objects slowly reorganize themselves.

The visual metaphor:

Understanding systems.

---

# Interaction

Hovering over the portrait slightly changes nearby nodes.

The portrait feels connected to the system.

Not placed on top of it.

---

# Scroll Transition

When scrolling downward,

the camera gently approaches the network.

Connections become clearer.

The visitor enters the Think chapter.

The transition should feel like moving into a workspace.

Not changing pages.

---

# Sound

None.

Silence is intentional.

---

# Typography

Headline dominates.

Whitespace dominates even more.

The interface should never feel crowded.

---

# Mobile

Portrait moves above the text.

Three.js scene becomes simpler.

Connections reduce.

Reading becomes the priority.

Navigation collapses.

No visual information is lost.

---

# Accessibility

Three.js never blocks interaction.

Content always remains selectable.

Reduced motion removes camera movement.

Keyboard users receive the same experience.

---

# Assets Required

Portrait

Transparent.

Hero background texture (optional)

Node texture

Connection shader

Social icons

Resume PDF

---

# Comments for CODEX

The Home screen defines the emotional tone for the entire portfolio.

Avoid trying to impress visitors.

Instead, create curiosity through simplicity.

Every visual decision should communicate thoughtfulness rather than spectacle.

The interface should feel crafted.

Not designed.

Not generated.

Crafted.
