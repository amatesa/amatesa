# 09 — Asset Guide

## Project

Personal Portfolio Website

Version 1.0

---

# Objective

Every visual asset must have a defined purpose before it exists.

No image, video or illustration should be created without knowing:

• why it exists

• where it will be used

• which dimensions it requires

• which visual style it should follow

• how it should be exported

This document acts as the visual production guide for every future asset.

---

# General Rules

Preferred formats

Photography → WebP

Illustrations → SVG

Icons → SVG

Video → MP4 (H.264)

Transparency → PNG or WebP Alpha

Never upload oversized assets.

Always export optimized versions.

# HERO

Purpose

Present Alejandro naturally.

Not as a corporate portrait.

Not as a LinkedIn photo.

The image should communicate curiosity and professionalism.

---

Expected filename

assets/images/hero/alejandro-hero.webp

Recommended resolution

1600 × 2000 px

Aspect ratio

4 : 5

Background

Transparent preferred.

Alternative:

Neutral background matching the portfolio palette.

---

Lighting

Soft.

Natural.

No dramatic shadows.

No colored lights.

No HDR effect.

---

Composition

Half body.

Looking slightly toward the page content.

Relaxed posture.

Neutral clothing.

No tie.

No strong branding.

No distracting objects.

---

Integration

The portrait should visually merge with the page.

No visible rectangular borders.

No card around the portrait.

No circular crop.

---
