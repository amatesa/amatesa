# 16 — Motion System

## Purpose

This document defines how movement behaves throughout the portfolio.

Motion is not decoration.

Motion is communication.

Every animation should help visitors understand relationships, hierarchy, interaction or progression.

If movement exists only because it looks impressive, it should be removed.

---

# Motion Philosophy

Visitors should never consciously notice the animations.

Instead, they should feel that the interface behaves naturally.

Movement should feel inevitable.

Objects respond.

They never perform.

---

# The Five Motion Principles

## 1. Motion Explains

Movement always communicates something.

Examples:

This belongs here.

This is connected.

This changed.

This is now active.

This became available.

---

## 2. Motion Respects Attention

Animations should never interrupt reading.

The visitor always controls the pace.

Nothing appears unexpectedly.

Nothing competes with the content.

---

## 3. Motion Has Mass

Every interface element behaves as if it has weight.

Cards don't pop.

They settle.

Panels don't appear.

They slide into place.

Objects don't disappear.

They return to the system.

---

## 4. Motion Has Intention

Every movement has a beginning.

A destination.

A reason.

There are no random transitions.

---

## 5. Motion Is Consistent

If one component behaves a certain way, every similar component follows the same rules.

Consistency creates trust.

---

# Motion Vocabulary

The portfolio only uses a small number of movement types.

## Fade

Used only when introducing information.

Never as the primary animation.

---

## Slide

Used when elements enter or leave the layout.

Preferred over fading.

---

## Scale

Used sparingly.

Objects scale only between 98% and 102%.

Never dramatic.

---

## Morph

Preferred whenever possible.

Buttons become panels.

Cards become workspaces.

Images become case studies.

Objects transform rather than disappear.

---

## Camera Movement

Reserved for major navigation moments.

Never for decorative purposes.

# Easing

Default

Ease Out Cubic

The interface should slow into place.

Never slam to a stop.

---

Interactive

Ease In Out

Used when objects move because of user interaction.

---

Camera

Custom Bezier

Acceleration should feel physical rather than mathematical.

---

# Scroll Behaviour

Scrolling should reveal.

Not trigger.

Sections appear because the visitor reaches them.

Not because scroll percentage reached a threshold.

---

# Hover Behaviour

Hover should suggest.

Not surprise.

Typical effects

Slight elevation.

Slight saturation increase.

Subtle shadow adjustment.

Tiny positional shift.

Never exaggerated scaling.

---

# Click Behaviour

Clicks should feel decisive.

Feedback appears immediately.

The resulting transition begins without delay.

The visitor never wonders whether the click worked.

---

# Three.js Philosophy

Three.js exists to visualize relationships.

Never spectacle.

Never decoration.

Never technical demonstration.

The visitor should think:

"I understand."

Not:

"That's impressive."

---

# Three.js Scene Rules

Every object represents information.

Every line represents a relationship.

Every movement represents change.

Nothing exists purely for atmosphere.

---

# Camera Language

The camera behaves like a curious observer.

It approaches.

Examines.

Steps back.

Reframes.

It never spins.

Never shakes.

Never rotates unnecessarily.

---

# Spatial Language

Objects belong to an invisible structure.

The scene should feel organized.

Not simulated.

Not chaotic.

Distance communicates hierarchy.

Alignment communicates understanding.

---

# Microinteractions

Microinteractions are rewards.

Examples:

A connection appears after reading.

A hidden note expands.

A project unfolds.

A concept links to another.

Each interaction teaches something.

---

# Mouse Behaviour

Cursor movement subtly influences nearby elements.

Never more than a few pixels.

The goal is responsiveness.

Not attraction.

---

# Keyboard Behaviour

Everything available by mouse remains fully accessible by keyboard.

Focus transitions receive the same design attention as hover states.

---

# Mobile Motion

Motion simplifies.

No unnecessary parallax.

No heavy camera movement.

Three.js scenes become lighter.

Interactions become shorter.

Reading becomes easier.

---

# Reduced Motion

If the operating system requests reduced motion:

Remove non-essential animation.

Keep spatial continuity.

Replace movement with opacity and layout changes.

Accessibility always overrides visual expression.

---

# Performance Budget

Motion should never reduce responsiveness.

Animation must remain smooth.

The interface should degrade gracefully on slower devices.

Content is always prioritized over effects.

# Success Criteria

Visitors should remember understanding.

Not animation.

If someone says:

"The website was beautiful."

The motion succeeded only partially.

If someone says:

"I understood how this person thinks."

The motion system has achieved its purpose.

---

# Final Principle

Movement should make ideas easier to understand.

Never harder to notice.
