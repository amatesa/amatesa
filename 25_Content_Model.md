# 25 — Content Model

## Purpose

This document defines every content entity supported by the portfolio.

The portfolio is not organized around pages.

It is organized around knowledge.

Every entity represents a different way Alejandro creates, learns, builds or shares ideas.

The interface is simply one possible way of presenting these entities.

---

# Core Philosophy

Everything is content.

Projects.

Ideas.

Books.

Experiments.

Courses.

Articles.

Talks.

Learning.

Nothing deserves its own architecture.

Everything follows the same design language.

---

# Comments for CODEX

Build a content-driven platform.

Do not build a website composed of pages.

Entities should remain independent from presentation.

Every relationship should be discoverable.

Every entity should be reusable.

The interface renders knowledge.

It never owns knowledge.

---

# Final Principle

The portfolio should evolve exactly as Alejandro evolves.

Adding new interests should expand the system.

Never require rebuilding it.
