# 13 — Design Principles

## Purpose

This document defines the principles that protect the identity of the portfolio.

Whenever a design or implementation decision is unclear, these principles take precedence over trends, personal preferences, or visual novelty.

The objective is consistency.

Not decoration.

---

# Principle 01

Understanding before aesthetics.

If a visual effect reduces clarity, remove it.

---

# Principle 02

Content is always more important than decoration.

Design exists to support ideas.

Never compete with them.

---

# Principle 03

Never imitate a portfolio trend.

The goal is to build something timeless.

Not something fashionable.

---

# Principle 04

Technology is evidence.

Not identity.

Visitors should remember ideas before frameworks.

---

# Principle 05

Every animation answers one question.

"What does this interaction communicate?"

If the answer is "nothing", remove it.

---

# Principle 06

Movement should guide attention.

Never steal attention.

---

# Principle 07

Whitespace is content.

Do not fill every empty area.

Silence is part of the reading experience.

---

# Principle 08

Every section should communicate one idea.

Not multiple ideas.

---

# Principle 09

Curiosity should drive exploration.

Never force exploration.

Visitors should want to continue.

---

# Principle 10

Complexity belongs inside the code.

Simplicity belongs in the interface.

---

# Principle 11

The portfolio should feel handcrafted.

Not assembled from a template.

---

# Principle 12

Every interaction should feel inevitable.

Nothing should appear arbitrary.

---

# Things We Never Do

Never use skill bars.

Never use percentage ratings.

Never use star ratings.

Never claim expertise without evidence.

Never exaggerate achievements.

Never hide academic work behind misleading wording.

Never autoplay sound.

Never use stock photography.

Never overload pages with text.

Never create animations for decoration only.

Never sacrifice accessibility.

Never sacrifice performance.

Never sacrifice readability.

Never place technologies before problems.

Never describe Alejandro through tools alone.

Never use generic marketing language.

Never use:

"Passionate professional"

"Results-driven"

"Dynamic team player"

"Hard worker"

"Self-motivated"

or similar clichés.

---

# If Two Solutions Are Equally Good

Choose the one that is:

Simpler.

Clearer.

More maintainable.

More accessible.

More honest.

---

# Long-Term Vision

This portfolio should improve over time without changing its identity.

New projects should feel like new chapters.

Not redesigns.

---

# Final Principle

If a future version becomes more visually impressive but less authentic, the redesign has failed.

If a future version becomes simpler while communicating Alejandro's way of thinking more clearly, it has succeeded.
