# 03 — Information Architecture

## Project

Personal Portfolio Website

Version 1.0

---

# Objective

The portfolio should not feel like reading a résumé.

It should feel like discovering a person's way of thinking through a sequence of carefully designed experiences.

The visitor should naturally move from curiosity to trust.

Navigation should feel effortless.

---

# Navigation Model

Single Page Application experience.

Static HTML implementation.

Each major section behaves as an independent scene.

Smooth scrolling connects every scene.

Navigation is persistent.

The active section is always visible.

# User Journey

Visitors should naturally answer five questions.

Who is this person?

↓

How does this person think?

↓

What has this person built?

↓

What is this person learning?

↓

How can I contact them?

Everything in the portfolio supports this sequence.

# Success Metrics

Visitors should:

Understand the purpose of the portfolio in under 15 seconds.

Reach the Build section.

Open at least one project.

Visit GitHub or Itch.io.

Download the résumé or contact Alejandro.

---

# Guiding Principle

Visitors should never wonder where to click next.

Every section should naturally invite exploration of the following one.

The experience should feel continuous, intentional and memorable.
