# 02 — Brand

## Project

Personal Portfolio Website

Version 1.0

---

# Brand Purpose

The portfolio should communicate the personality of a builder.

Someone who enjoys understanding systems, experimenting with ideas and continuously improving them.

The visual identity should reflect curiosity supported by structure.

Creativity should emerge through interaction and composition rather than decoration.

---

# Brand Attributes

The portfolio should be perceived as:

• Modern

• Minimal

• Intelligent

• Calm

• Creative

• Precise

• Experimental

• Elegant

It should never feel:

• Corporate

• Generic

• Loud

• Futuristic for its own sake

• Dark gamer

• Portfolio template

---

# Personality

Imagine the portfolio as a person.

Curious.

Thoughtful.

Patient.

Always observing.

Always improving.

Never trying to impress.

Never pretending to know everything.

Confidence comes from clarity.

Not from exaggeration.

---

# Tone of Voice

Writing should be:

Clear.

Short.

Direct.

Reflective.

Professional.

Avoid:

Marketing language.

Buzzwords.

Self-promotion.

Empty adjectives.

Claims without evidence.

The portfolio should demonstrate value instead of declaring it.

---

# Visual Language

Everything should communicate precision.

Large margins.

Consistent rhythm.

Clean typography.

Soft shadows.

Few colors.

Large photography.

Simple icons.

Thoughtful spacing.

Nothing decorative without purpose.

---

# Color Philosophy

Color is used to guide attention.

Not to decorate.

One primary accent color.

Neutral surfaces.

Minimal gradients.

Never use multiple competing accent colors.

---

# Primary Accent

Deep Teal

HEX

#174349

The accent color should represent:

growth

exploration

stability

technology

calmness

---

# Neutral Palette

Create an entire neutral scale.

Very light backgrounds.

Soft grays.

Warm whites.

Dark charcoal.

Avoid:

Pure black

Pure white

Strong saturated colors

---

# Theme Support

Two complete themes.

Light.

Dark.

Both should have equal design quality.

Dark mode must not simply invert colors.

Every color should be intentionally selected.

---

# Typography

Typography is one of the primary visual elements.

Use only one font family.

Preferred order:

Inter

Manrope

IBM Plex Sans

System UI fallback

Hierarchy should come from:

weight

size

spacing

Never rely on color alone.

---

# Photography

Photography should feel integrated into the interface.

Never look like a rectangular image placed inside a page.

The portrait should blend naturally with the surrounding composition.

Soft masks.

Background integration.

Subtle lighting.

No visible borders.

No heavy frames.

No circular avatars.

---

# Illustrations

Avoid decorative illustrations.

Instead use:

diagrams

timelines

processes

simple geometric compositions

interactive visual explanations

---

# Iconography

Outline icons.

Consistent stroke width.

Simple.

Minimal.

No colorful icon packs.

Prefer SVG.

---

# Components

Every component should feel related.

Shared spacing.

Shared radius.

Shared motion.

Shared typography.

No isolated design styles.

---

# Motion Identity

Movement communicates understanding.

Not entertainment.

Animations should feel:

smooth

natural

predictable

purposeful

Transitions should suggest that information is unfolding rather than appearing.

---

# Interaction Style

Interactions should reward curiosity.

Hover states reveal information.

Cards may expand.

Sections may transform.

Navigation should feel exploratory.

Avoid repetitive animation patterns.

Every major interaction should have a reason.

---

# Layout Philosophy

Large breathing spaces.

Strong alignment.

Clear visual hierarchy.

Asymmetric compositions when appropriate.

Content should never feel crowded.

---

# Gamer Influence

Game inspiration should appear through interaction design.

Never through visual clichés.

Avoid:

pixel art

controllers

RGB

cyberpunk

fantasy

swords

Instead borrow ideas from games such as:

exploration

progression

discovery

feedback

micro interactions

environment transitions

layered information

---

# Emotional Progression

Visitors should experience:

Curiosity

↓

Understanding

↓

Confidence

↓

Interest

↓

Trust

↓

Conversation

---

# Accessibility

Beauty never overrides usability.

Every interaction must remain accessible.

Keyboard navigation.

Visible focus states.

Readable typography.

High contrast.

Reduced motion support.

---

# Brand Promise

Every page should communicate one idea:

This is the workspace of someone who enjoys understanding complex problems and building thoughtful digital solutions.

Everything else is secondary.
