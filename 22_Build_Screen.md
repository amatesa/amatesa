# Screen 22 — Build

## Purpose

Build demonstrates how ideas become execution.

This chapter is not a project gallery.

It is a workshop.

Visitors should feel they are walking among real work rather than browsing portfolio cards.

Every project exists because it solved a problem.

Technology is evidence.

Problem solving is the story.

---

# UX Goal

Visitors should naturally conclude:

"This person enjoys building systems."

Not:

"This person knows many technologies."

Projects demonstrate thinking.

Not tools.

---

# Narrative

Think answered:

"How does Alejandro think?"

Build answers:

"What happens when that way of thinking becomes work?"

Every project reinforces the same process.

Observe.

Understand.

Design.

Build.

Improve.

---

# Emotional Goal

Confidence.

Craftsmanship.

Curiosity.

Credibility.

The visitor should begin imagining Alejandro working inside a team.

---

# Scene Composition

The workspace becomes larger.

The visitor is no longer observing ideas.

The visitor is surrounded by work.

Objects occupy a conceptual workbench.

Nothing floats randomly.

Everything has purpose.

# Accessibility

Every project remains fully accessible without Three.js.

The Project Workspace exists as semantic HTML.

Keyboard users receive identical navigation.

Reduced motion replaces camera movement with layout transitions.

---

# Performance

Project media loads lazily.

Three.js geometry remains lightweight.

No project should delay interaction.

Prioritize content before media.
