## Project

# Objective

Content is the product.

Design exists to support content.

Every section should answer a specific question.

No filler text.

No lorem ipsum.

No generic portfolio language.

---

# Configuration Philosophy

No section should require deleting code.

Visibility should be controlled through configuration flags.

Developers should be able to activate future content by changing a single value.

---

# Content Maintenance

Every page includes developer comments indicating:

Missing assets

Recommended replacements

Pending copy

Future improvements

Image optimization reminders

Accessibility reminders

SEO reminders

---

# Final Principle

The repository should be self-documenting.

A developer opening the project for the first time should immediately understand:

What belongs here.

What is missing.

What format is expected.

How every asset should be replaced.
